## 1. 日常维护原则

这个网站采用“Markdown 写作 + 自动生成网页”的方式维护。

你平时只需要关注：

- 文章源文件：`content/posts/*.md`
- 一键新建文章：`npm run new:post`
- 一键发布网站：`npm run publish`

不要手工修改 `posts/*.html`。这些 HTML 文件是由脚本自动生成的，下一次构建会被覆盖。

## 3. 新增一篇文章

推荐使用这个命令：

```powershell
npm run new:post -- -Title "文章标题" -Category "工作" -Summary "首页副标题" -Keywords "关键词1 关键词2" -Open
```

示例：

```powershell
npm run new:post -- -Title "如何把周例会开得更短" -Category "工作" -Summary "从议题、材料到纪要" -Keywords "会议 周例会 纪要" -Open
```

参数说明：

- `-Title`：文章标题，必填
- `-Category`：分类，建议使用 `工作`、`生活`、`读书`、`复盘`
- `-Summary`：首页列表里显示的小字说明，可以留空
- `-Keywords`：搜索关键词，多个词用空格隔开
- `-Open`：创建后自动打开 Markdown 文件

生成后的文章会放在：

```text
content/posts/
```

## 4. Markdown 文章格式

每篇文章开头必须有这段信息：

```markdown
---
title: 文章标题
date: 2026-06-13
category: 工作
summary: 首页副标题，可留空
keywords: 搜索关键词
---

这里写正文。
```

注意：

- `date` 必须是 `YYYY-MM-DD` 格式
- `category` 会决定首页标签和筛选按钮
- `summary` 会显示在首页标题后面
- `keywords` 会进入首页搜索

正文支持常用 Markdown：

```markdown
## 二级标题

普通段落。

- 列表项一
- 列表项二

> 引用内容

`行内代码`
```

## 5. 用 Obsidian 写文章

推荐方式是让 Obsidian 只负责写 Markdown，网站负责生成 HTML。

Obsidian 写作目录：

```text
D:\Nutstore\Obsidian Vault\posts
```

网站文章源目录：

```text
D:\Nutstore\0.2026年-代办工作\work-life-reflections-promo\website\content\posts
```

注意：不要同步或手工编辑下面这个目录：

```text
D:\Nutstore\0.2026年-代办工作\work-life-reflections-promo\website\posts
```

这里是自动生成的 HTML 文章页，不是源文件。

第一次使用 Obsidian 前，先把网站现有文章复制到 Obsidian：

```powershell
cd "D:\Nutstore\0.2026年-代办工作\work-life-reflections-promo\website"
npm run obsidian:init
```

以后你在 Obsidian 写完或修改文章后，运行：

```powershell
cd "D:\Nutstore\0.2026年-代办工作\work-life-reflections-promo\website"
npm run obsidian:publish
```

它会自动完成：

- 从 Obsidian 复制新增或修改过的 Markdown
- 重新生成网页
- 提交到 Git
- 推送到 GitHub

如果只想同步并本地预览，不想发布：

```powershell
npm run obsidian:sync
```

同步脚本不会自动删除文件，避免误删文章。需要删除文章时，请同时删除 Obsidian `posts` 和网站目录 `content/posts/` 里的同名 Markdown，再运行 `npm run publish`。

### 自动发布模式

如果希望“Obsidian 写完保存后自动上传网站”，安装后台监听任务：

```powershell
cd "D:\Nutstore\0.2026年-代办工作\work-life-reflections-promo\website"
npm run obsidian:auto-install
```

安装后：

- Windows 登录时会自动启动监听程序
- 监听目录是 `D:\Nutstore\Obsidian Vault\posts`
- 保存 Markdown 后，等待 60 秒无新变化，会自动发布
- 自动发布内容包括：同步 Markdown、生成 HTML、提交 Git、推送 GitHub

如果想临时在前台查看监听过程，可以运行：

```powershell
npm run obsidian:watch
```

自动发布日志在：

```text
%LOCALAPPDATA%\sgecswtp-site\obsidian-autopublish.log
```

如果以后想关闭自动发布：

```powershell
npm run obsidian:auto-remove
```

注意：自动发布要求电脑在线，并且本机 GitHub 登录状态正常。如果 GitHub 网络不可用，日志里会记录失败原因，下一次保存文章时会再次尝试。

发布脚本会重新生成首页和文章页，旧的 `posts/*.html` 会自动清理。

## 11. 常见问题

### 文章没有出现在首页

检查：

- 文件是否放在 `content/posts/`
- 文件扩展名是否是 `.md`
- 开头是否有 `---` front matter
- `date` 是否是 `YYYY-MM-DD`
- 是否运行了 `npm run build` 或 `npm run publish`

### 发布后网站没有立刻变化

GitHub Pages 需要一点时间构建。通常等待 1 到 3 分钟。

也可以打开 GitHub 仓库的 `Actions` 页面，查看最新工作流是否成功。

### `npm run publish` 提示没有变化

这说明本地没有新内容需要发布。可以先运行：

```powershell
git status
```

确认是否真的修改了文章文件。

### Git push 失败

常见原因：

- 网络无法连接 GitHub
- GitHub 登录过期
- 本地和远程有冲突

可以先运行：

```powershell
git status
git pull
```

如果仍然失败，把错误信息发给 Codex。
