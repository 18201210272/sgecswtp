适用网站：`sgecswtp.top`  
本地目录：`D:\Nutstore\0.2026年-代办工作\work-life-reflections-promo\website`  
GitHub 仓库：`https://github.com/18201210272/sgecswtp`

## 1. 日常维护原则

这个网站采用“Markdown 写作 + 自动生成网页”的方式维护。

你平时只需要关注：

- 文章源文件：`content/posts/*.md`
- 一键新建文章：`npm run new:post`
- 一键发布网站：`npm run publish`

不要手工修改 `posts/*.html`。这些 HTML 文件是由脚本自动生成的，下一次构建会被覆盖。

## 3. 新增一篇文章

推荐使用这个命令：

```powershell
npm run new:post -- -Title "文章标题" -Category "工作" -Summary "首页副标题" -Keywords "关键词1 关键词2" -Open
```

示例：

```powershell
npm run new:post -- -Title "如何把周例会开得更短" -Category "工作" -Summary "从议题、材料到纪要" -Keywords "会议 周例会 纪要" -Open
```

参数说明：

- `-Title`：文章标题，必填
- `-Category`：分类，建议使用 `工作`、`生活`、`读书`、`复盘`
- `-Summary`：首页列表里显示的小字说明，可以留空
- `-Keywords`：搜索关键词，多个词用空格隔开
- `-Open`：创建后自动打开 Markdown 文件

生成后的文章会放在：

```text
content/posts/
```

## 4. Markdown 文章格式

每篇文章开头必须有这段信息：

```markdown
---
title: 文章标题
date: 2026-06-13
category: 工作
summary: 首页副标题，可留空
keywords: 搜索关键词
---

这里写正文。
```

注意：

- `date` 必须是 `YYYY-MM-DD` 格式
- `category` 会决定首页标签和筛选按钮
- `summary` 会显示在首页标题后面
- `keywords` 会进入首页搜索

正文支持常用 Markdown：

```markdown
## 二级标题

普通段落。

- 列表项一
- 列表项二

> 引用内容

`行内代码`
```

## 8. 发布网站

最简单发布命令：

```powershell
npm run publish
```

它会自动完成：

1. 生成首页和文章页
2. 检查是否有变化
3. `git add .`
4. `git commit`
5. `git push`

推送成功后，GitHub Pages 会自动更新网站。

通常等待 1 到 3 分钟后访问：

```text
https://sgecswtp.top
```

## 10. 删除文章

删除对应 Markdown 文件：

```text
content/posts/某篇文章.md
```

然后运行：

```powershell
npm run publish
```

注意：当前脚本会重新生成首页，但旧的 `posts/*.html` 可能还留在本地。需要彻底删除旧文章页时，可以手工删除对应的 `posts/文章名.html` 后再发布。

## 11. 常见问题

### 文章没有出现在首页

检查：

- 文件是否放在 `content/posts/`
- 文件扩展名是否是 `.md`
- 开头是否有 `---` front matter
- `date` 是否是 `YYYY-MM-DD`
- 是否运行了 `npm run build` 或 `npm run publish`

### 发布后网站没有立刻变化

GitHub Pages 需要一点时间构建。通常等待 1 到 3 分钟。

也可以打开 GitHub 仓库的 `Actions` 页面，查看最新工作流是否成功。

### `npm run publish` 提示没有变化

这说明本地没有新内容需要发布。可以先运行：

```powershell
git status
```

确认是否真的修改了文章文件。

### Git push 失败

常见原因：

- 网络无法连接 GitHub
- GitHub 登录过期
- 本地和远程有冲突

可以先运行：

```powershell
git status
git pull
```

如果仍然失败，把错误信息发给 Codex。
